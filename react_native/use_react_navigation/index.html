React Navigation を使って画面遷移を実装する（Expo）

前回 作成したTODOアプリのフォーム画面と一覧表示画面分けてみましょう。

React Native で画面遷移機能を実装するためにいくつかパッケージがあります。ここでは最もよく使われる React Navigation を使用して画面遷移の機能を追加していきます。React Navigation のバージョンは５系のものを利用します。

また今回の完成済のコードは こちら です。

React Navigation のインストール

まずは React Navigation 本体をインストールしましょう。 

    
      
$ yarn add @react-navigation/native

次に依存関係のある必要パッケージをインストールします。 このチュートリアルでは Expo を使用してプロジェクトを作成しているので、 これらのパッケージは expo install でインストールする必要があります。 Expo を利用していない場合は yarn または npm でインストールするようにしましょう。 

    
      
$ yarn expo install react-native-gesture-handler react-native-reanimated react-native-screens react-native-safe-area-context @react-native-community/masked-view

さて最後に @react-navigation/stack をインストールします。 これはスタックして画面遷移する機能を提供してくれるパッケージです。 「スタックして」とは、通常のアプリでよくある遷移で、遷移先の画面が横スライドしながら遷移元画面の上に表示される動きをみせます。 画面下タブで画面を切り替える場合やハンバーガーメニューのような切り替えをする場合は別のパッケージが必要となります。 

    
      
$ yarn add @react-navigation/stack

以上でパッケージのインストールは完了です。

ホーム画面の実装

基本的な実装の流れは、１画面単位で画面を描画するコンポーネントを作成していきます。 作成したコンポーネントをナビゲーターというものに登録します。 この時画面名も同時に登録します。 あとは画面遷移するタイミングで画面登録した名前を呼び出すだけです。

シンプルな構成で、実際機能を作るのは非常に楽です。 では最初にToDoの一覧を表示する画面を作成してみましょう。

管理しやすくするために画面用のコンポーネントは src/screens 配下に作るようにします。 src/screens/Home.jsというファイルを作成し、前回作った App.js から一覧表示の機能部分だけ抜き出します。

src/screens/Home.js
    
      
import React, { Component } from 'react';
import {
  SafeAreaView,
  FlatList,
  View,
  Text,
  StyleSheet
} from 'react-native';

/* テストデータ */
const todoList = [
  { todo: '買い物', memo: '今日の午後６時まで' },
  { todo: '掃除', memo: '今日は必ずやる' }
];

export default class Home extends Component {
  render() {
    return (
      <SafeAreaView style={styles.container}>
        <FlatList
          style={styles.listBox}
          data={todoList}
          renderItem={({ item }) => {
            return (
              <View style={styles.listItem}>
                <Text>{item.todo}</Text>
                <Text>{item.memo}</Text>
              </View>
            );
          }}
          keyExtractor={item => item.todo}
        />
      </SafeAreaView>
    );
  }
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    flexDirection: 'column',
    backgroundColor: '#fff',
    alignItems: 'center',
    justifyContent: 'center',
  },
  listItem: {
    height: 64,
    width: 200,
    marginTop: 16,
    padding: 16,
    borderColor: 'gray',
    borderWidth: 1,
  }
});

前回の内容の部分なので特に説明は不要でしょう。

次にこの一覧表示画面をアプリのホーム画面として表示されるように App.js を修正します。

App.js
    
      
import React, { Component } from 'react';
import { NavigationContainer } from '@react-navigation/native';
import { createStackNavigator } from '@react-navigation/stack';
import Home from './src/screens/Home';

const Stack = createStackNavigator();

export default class App extends Component {

  render() {
    return (
      <NavigationContainer>
        <Stack.Navigator>
          <Stack.Screen name="Home" component={Home} />
        </Stack.Navigator>
      </NavigationContainer>
    );
  }
}

NavigationContainerは遷移の構成や状態など全てを管理するコンポーネントです。 画面のコンポーネントは全て NavigationContainer に内包されるようにして置く必要があります。

createStackNavigatorはScreenとNavigator の２つのプロパティを持つオブジェクトを返す関数です。 Navigator は Screen を含むようにしておき、 Screen の属性には、 nameに画面の名前、 componentにはその画面を描画するコンポーネントを設定します。

ここまで完了したらアプリを実行してください。すると画面上にヘッダーが表示されるようになります。

画面遷移の実装

さてメインの画面遷移の実装に入ります。まずはForm画面の切り出しをしておきます。

src/screens/Form.js
    
      
import React, { Component } from 'react';
import {
  SafeAreaView,
  Button,
  StyleSheet
} from 'react-native';
import FormInput from '../components/FormInput';

export default class Form extends Component {
  constructor(props) {
    super(props);
    this.state = {
      todoValue: "",
      memoValue: "",
      todoList: [],
    };
  }

  render() {
    const { todoValue, memoValue, todoList } = this.state;

    return (
      <SafeAreaView style={styles.container}>
        <FormInput
          label="やること"
          value={todoValue}
          placeholder="何かやること"
          onChangeText={v => this.setState({ todoValue: v })}
        />
        <FormInput
          label="メモ"
          value={memoValue}
          placeholder="何かメモ"
          onChangeText={v => this.setState({ memoValue: v })}
        />
        <Button
          title="登録"
          onPress={() => {
            const newList = todoList.concat({ todo: todoValue, memo: memoValue });
            this.setState({
              todoValue: "",
              memoValue: "",
              todoList: newList
            });
          }}
        />
      </SafeAreaView>
    );
  }
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    flexDirection: 'column',
    backgroundColor: '#fff',
    alignItems: 'center',
    justifyContent: 'center',
  }
});

この画面自体の内容も前回のものなので説明は割愛します。

Navigator の initialRouteName 属性にはアプリ起動時に最初に表示される画面を設定します。 未設定の場合は Navigator の一番上に書かれている Screen が最初に表示されます。

Screen に登録したコンポーネントの props には自動で navigationという props が挿入されるようになっています。

navigationにはnavigateという画面へ移動する関数が用意されています。 この関数の引数に移動さきの画面名を設定するとその画面へ遷移する機能が動作します。

補足ですが、ヘッダーには今「Home」「Form」と登録した画面名（Screen の name 属性の値）が表示されるようになっています。

データの受渡しについて

これで画面遷移についての実装は完了しました。 と言いたいとことですが、お気づきのように今の実装ではToDo登録画面から登録しても結果が反映されません。

これについては２通りの方法があります。

１つは情報は全てAppコンポーネントのStateで管理し、 各画面コンポーネントのPropsに詰めるようにしてデータのやりとりをします。

もう１つはContextというReact Nativeに用意されているデータ管理の機能を使う方法です。

ここでは１つ目の方法で機能を追加していきます。

まずは、App.jsを下記のように編集して App.jsのStateとStateを更新する関数を各画面へ渡すようにします。

App.js
    
      
export default class App extends Component {
  constructor(props) {
    super(props)
    this.state = {
      todoList: [],
    };
  }

  addNewItem(todoList) {
    this.setState({ todoList })
  }

  render() {
    const { todoList } = this.state;
    return (
      <NavigationContainer>
        <Stack.Navigator initialRouteName="Home">
          <Stack.Screen
            name="Home"
            options={ { title: '一覧' } }
          >
            {props => <Home {...props} todoList={todoList} />}
          </Stack.Screen>
          <Stack.Screen
            name="Form"
            options={ { title: '登録' } }
          >
            {props => <Form {...props} todoList={todoList} addNewItem={v => this.addNewItem(v)} />}
          </Stack.Screen>
        </Stack.Navigator>
      </NavigationContainer>
    );
  }
}

Stack.Screenのcomponent属性を削除し、 Stack.Screenの子要素に直接画面コンポーネントを書くようにします。 子要素は画面コンポーネントをそのまま使わず、Propsを受け取ってから画面コンポーネントを返す関数にしておきます。

この時、画面コンポーネントにはPropsとApp.jsから渡す要素を設定しておきます。 この方法でAppコンポーネントのStateを各画面に共有することができます。

あとは画面の方はAppコンポーネントから渡された値を利用するように変更するだけです。

src/screens/Form.js
    
      
export default class Form extends Component {
  constructor(props) {
    super(props);
    this.state = {
      todoValue: "",
      memoValue: "",
    };
  }

  render() {
    const { todoValue, memoValue } = this.state;
    const { navigation, todoList } = this.props;

    return (
        ...

        <Button
          title="登録"
          onPress={() => {
            const newList = todoList.concat({ todo: todoValue, memo: memoValue });
            this.setState({
              todoValue: "",
              memoValue: "",
            });
            this.props.addNewItem(newList);
            navigation.goBack();
          }}
        />

        ...
    );
  }
}

各画面を編集したら登録画面からToDoを登録してみてください。 すると登録した内容が一覧画面の方へ追加させているのが確認できるでしょう。
